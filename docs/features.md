# 마녀봇 기능 목록

## 봇 명령어 (유저용)

### 재화 관리
```
@봇 내재화
```
- 현재 보유 재화 확인
- 누적 획득/사용 재화 확인
- DM으로 응답

### 상점
```
@봇 상점
```
- 구매 가능한 아이템 목록
- 카테고리별 분류
- 가격 표시

```
@봇 구매 [아이템명]
```
- 아이템 구매
- 재화 차감
- inventory에 추가
- 재화 부족 시 실패

```
@봇 내아이템
```
- 보유 아이템 목록
- 획득 날짜 표시

### 휴식 관리
```
@봇 휴식 N
```
- 오늘부터 N일간 휴식 등록
- 예: `@봇 휴식 2` → 오늘부터 2일간 (오늘, 내일)
- 예: `@봇 휴식 5` → 오늘부터 5일간
- 자동 등록 (관리자 승인 불필요)
- 기간 동안 관리자 웹에서 💤 휴식 중으로 표시됨

```
@봇 휴식 해제
```
- 현재 활성화된 휴식 즉시 종료
- 관리자 웹에서 휴식 상태 즉시 해제됨

### 게임 (추후 구현)
- 봇 대화형 게임 시스템
- 재화를 걸고 승부
- 구체적인 게임 종류는 추후 결정

### 일정 조회
```
@봇 일정
```
- 이번 주 커뮤니티 일정 조회 (오늘 ~ 7일 이내)
- 다가오는 이벤트 표시
- 전역 휴식기간 안내
- 공휴일/특별 이벤트 표시

```
@봇 일정 이번달
```
- 이번 달 전체 일정 조회

```
@봇 일정 다음주
```
- 다음 주 일정 조회 (7일 ~ 14일 이내)

### 도움말
```
@봇 도움말
```
또는
```
@봇 help
```
- 사용 가능한 모든 명령어 목록
- 각 명령어별 간단한 사용법
- 예시 포함
- DM으로 응답

**응답 예시:**
```
🤖 봇 명령어 도움말

[재화 및 아이템]
• @봇 내재화 - 보유 재화 조회
• @봇 상점 - 아이템 목록
• @봇 구매 [아이템명] - 아이템 구매
• @봇 내아이템 - 보유 아이템 조회

[휴식 관리]
• @봇 휴식 N - N일간 휴식 등록
• @봇 휴식 해제 - 휴식 해제

[정보 조회]
• @봇 일정 - 이번 주 일정
• @봇 공지 - 최신 공지 및 안내
• @봇 도움말 - 이 도움말 표시
```

### 공지 조회
```
@봇 공지
```
- 최신 공지사항 표시 (최근 3개)
- 중요 안내사항 표시
- 다가오는 일정/이벤트 (7일 이내)
- 전역 휴식기간 안내
- 진행 중인 이벤트 안내
- DM으로 응답

**응답 예시:**
```
📢 공지 및 안내

[최신 공지]
• 2025-11-20: 연말 이벤트 안내
• 2025-11-18: 활동량 기준 변경 안내
• 2025-11-15: 신규 아이템 출시

[다가오는 일정] (7일 이내)
• 11/30 (토) 🎉 커뮤니티 모임
• 12/03 (화) 📢 중요 공지 예정
• 12/25 (수) 🏖️ 크리스마스 휴식기간 (출석 트윗 없음)

[현재 진행 중]
• 출석 연속 30일 달성 이벤트

자세한 내용은 @봇 일정 또는 관리자 웹에서 확인하세요.
```

## 자동 시스템

### 재화 지급 (일괄 정산)
- **정산 시간**: 하루 2회 (오전 4시, 오후 4시)
- **계산 방식**: 이전 정산 이후 ~ 현재까지 작성한 답글 수 집계
- **지급 공식**: `(답글 수 / N) * M원` (예: 100개당 10원)
- **중복 방지**: transactions 테이블에 기록된 답글 제외
- **장점**:
  - 비율이 높을 때 효율적 (100개당 10원 등)
  - 일괄 정산으로 PostgreSQL 접근 최소화
  - DB 부하 최소화
- 모든 거래 DB 기록

### 출석 체크 (하루 1회, Asia/Seoul 타임존 기준)
- **자동 출석 트윗 발행**: 매일 오전 10시(서울 시간)에 봇이 출석 트윗 자동 게시
- **답글로 출석**: 유저가 출석 트윗에 답글을 달면 자동으로 재화 지급
- **보상**: 기본 출석 50원 (고정)
- **중복 방지**:
  - DB 레벨: UNIQUE(user_id, attendance_post_id) 제약
  - 같은 출석 트윗에 여러 번 답글 달아도 1회만 인정
- **시간 제한**: 출석 트윗 게시 후 23시간 59분까지 유효
- **휴식계 처리**: 휴식 중인 유저도 출석 가능 (활동량 체크와 별개)
- **출석 기록**: 모든 출석은 attendances 테이블에 기록 (reward_amount 저장)
- **전역 휴식기간**:
  - calendar_events에서 is_global_vacation=true인 날짜 확인
  - 해당 날짜에는 **출석 트윗 발행 자체를 막음** (cron/celery에서 체크)

### 유저 등록

**팔로우 이벤트 기반 (실시간, 주요 방식)**
- **트리거**: 신규 유저가 어드민 계정 팔로우
- **처리**: Streaming API 이벤트 → 즉시 DB 등록
- **장점**:
  - ⚡ 실시간 등록 (지연 제로)
  - 🎯 가입 의사 명확 (팔로우 = 참여)
  - 💪 오버헤드 없음 (이벤트 기반)
- **환영 메시지**: 선택적으로 환영 DM 발송 가능
- **언팔로우**: DB 삭제 안 함 (재화, 인벤토리 보존)

**Lazy Creation (백업 방식)**
- **목적**: 팔로우 이벤트 누락 시 대비 (Streaming API 다운타임 등)
- **트리거**: 답글 재화 지급 시 유저 없음
- **처리**: PostgreSQL 조회 → 즉시 생성
- **Redis 캐시**: 중복 체크 최소화 (1시간 TTL)

### 활동량 조회 (관리자 웹 전용)
- **자동 체크 없음**: 자동 경고 발송 시스템 없음
- **조회 전용**: 관리자가 개인별 분석 페이지에서 실시간 조회
- **조회 항목**:
  - 48시간 내 답글 수 확인 (PostgreSQL 실시간 조회)
  - 기준 대비 충족 여부 (기본: 20개)
  - 휴식 상태 확인
- **수동 경고**: 관리자가 필요 시 수동으로 경고 발송 (관리자 웹에서)

## 관리자 웹 기능

### 대시보드
- 전체 통계 (유저 수, 총 재화, 48h 답글)
- 오늘의 알림 (경고 유저, 휴식 중)
- **경고 누적 알림**
  - 🔴 경고 3회 이상 유저 수 (아카이빙 대상)
  - 🟡 경고 2회 유저 수 (주의 필요)
  - 📊 경고 추이 (일주일 통계)
- 봇 상태 (정상/오류, 마지막/다음 체크)
- 전체 유저 활동 현황 테이블
  - 🟢 정상 (기준 충족)
  - 🟡 주의 (기준의 70~99%)
  - 🔴 경고 (기준 미달)
  - 💤 휴식 중
  - ⚠️ 경고 누적 중 (경고 횟수 표시)

### 활동량 관리
- **개인별 활동량 조회** (실시간)
  - 유저 선택 → 개인별 분석 페이지
  - 48시간 답글 수 조회 (PostgreSQL)
  - 기준 대비 충족 여부 표시
  - 🟢 정상 / 🔴 기준 미달 / 💤 휴식 중
  - 수동 경고 발송 버튼 제공
- **경고 이력 조회**
  - 유저별 누적 경고 횟수 표시
  - 정렬: 경고 많은 순/최근 경고 순
  - 필터: 경고 1회 이상/2회 이상/3회 이상
  - 상세 정보: 최근 경고 내역, 경고 유형별 통계
  - 📦 아카이빙 대상 표시 (3회 이상)
- **수동 경고 발송**
  - 유저 선택
  - 메시지 작성 (템플릿 제공)
  - 확인 팝업
  - DM 발송 + warnings 기록
- **휴식 관리**
  - 휴식 중 유저 목록
  - 휴식 등록 (시작일, 종료일, 사유)
  - 휴식 해제

**참고:**
- 자동 경고 시스템 없음
- 모든 경고는 관리자가 수동으로 발송
- 활동량은 실시간 조회만 (DB 저장 안 함)

### 재화 관리
- 유저별 재화 현황
  - 정렬 (재화 많은순/적은순)
  - 필터 (역할/기숙사)
  - 검색 (유저명)
- 재화 조정
  - 지급/차감
  - 금액 입력
  - 사유 입력 (필수)
  - 확인 팝업
  - transactions + admin_logs 기록

### 상점 관리
- 아이템 목록
  - 카테고리 필터
  - 상태 필터 (판매중/중단)
- 아이템 추가
  - 이름, 설명, 가격
  - 카테고리 (기존 선택 or 신규 생성)
  - 이미지 URL (선택)
  - 판매 상태
- 아이템 수정/삭제
  - 가격 변경
  - 판매 중단/재개
  - 삭제 (확인 팝업)

### 일정/이벤트 관리
- **일반 이벤트 등록**
  - 커뮤니티 모임, 행사, 기념일 등
  - 제목, 설명, 날짜
  - 이벤트 타입 선택: event(일반 이벤트), holiday(공휴일), notice(공지)
  - `@봇 일정` 명령어로 모든 유저가 조회 가능
  - `@봇 공지` 명령어에서도 다가오는 이벤트 표시
- **전역 휴식기간 설정**
  - 일정 등록 시 "전역 휴식기간" 체크박스 선택
  - 해당 날짜에는 출석 트윗 발행 자동 비활성화
  - 예: 크리스마스, 신년 등 커뮤니티 전체 휴식일 지정
  - 유저들이 일정 조회 시 휴식기간임을 명확히 표시 (🏖️)
- **일정 목록**
  - 월별 캘린더 보기
  - 주별 리스트 보기
  - 필터 (이벤트 타입별, 전역 휴식기간만 보기)
- **일정 수정/삭제**
  - 기존 일정 편집
  - 삭제 (확인 팝업)

### 시스템 설정

**활동량 조회 설정**
- 조회 기준
  - 체크 기간 (시간) (기본: 48시간)
  - 최소 답글 수 (기본: 20개)
- **참고**: 자동 체크 시스템 없음, 관리자 웹에서 실시간 조회 전용

**재화 지급 설정**
- 답글 재화 일괄 정산
  - 재화 지급 기준 답글 수 (N개) (기본: 100)
  - N개당 지급할 재화량 (M원) (기본: 10)
  - 예: 100개당 10원

**출석 체크 설정**
- 출석 트윗 발행 시간 (기본: 10:00)
- 기본 출석 보상 (기본: 50원)
- 시스템 온/오프
  - 출석 체크 시스템 활성화/비활성화

**휴식 관리 설정**
- 최대 휴식 기간 (일) (기본: 90일)
- 봇 명령어 허용 여부
  - 유저 셀프 등록 허용/불허

**소셜 분석 설정**
- 고립 판정 기준
  - 대화 상대 N명 미만 (기본: 7명)
- 편중 판정 기준
  - 특정 1명과의 대화 비율 (기본: 30% = 0.3)
- 비활동 판정 기준
  - 7일 접속률 N% 미만 (기본: 50% = 0.5)

**경고 및 아카이빙 설정**
- 툿 아카이빙 임계값 (기본: 3회)
  - 경고 N회 이상 시 자동 아카이빙
- 어드민 계정 ID
  - 팔로우 감지용 마스토돈 계정 ID

**타임존 설정**
- 타임존 (기본: Asia/Seoul)

**설정 관리**
- 모든 설정 DB 저장 (system_config 테이블)
- 변경 즉시 적용
- 기본값 복원 기능
- 설정 변경 이력 (admin_logs)

### 커뮤니티 건강도 (소셜 분석)
- **대시보드 위젯**
  - 🔴 대화 편중 유저 수
  - 🟡 고립 위험 유저 수
  - 🟢 건강한 유저 수

- **편중 유저 목록**
  - 특정 1명과의 대화 > 30% 유저
  - 대화 상대 수, 최다 대화 상대, 비율 표시
  - 예: "@user1 → @user2와 40% (20/50개)"

- **고립 유저 목록**
  - 48시간 내 대화 상대 < 7명
  - 왕따 위험 유저 조기 발견

- **비활동 유저 목록**
  - 7일 접속률 < 50%
  - 이탈 위험 유저 파악

- **유저 상세 소셜 통계**
  - 대화 상대 분석 (48시간)
  - 접속 패턴 (7일)
  - 경고 이력

- **유저별 시각적 분석 페이지**
  - Chart.js 활용 시각화
  - **대화 패턴 차트**
    - 최근 7일 대화 상대 수 추이 (선 그래프)
    - 대화 상대별 답글 수 분포 (파이 차트)
    - 일별 총 답글 수 (막대 그래프)
  - **접속률 차트**
    - 최근 30일 활동 일수 (히트맵 캘린더)
    - 주간 접속률 추이 (선 그래프)
  - **건강도 지표**
    - 대화 다양성 점수 (0-100점)
    - 활동 점수 (0-100점)
    - 종합 건강도 (색상 코드: 🟢 건강/🟡 주의/🔴 위험)
  - **경고 이력 타임라인**
    - 시간순 경고 내역
    - 경고 유형별 통계

- **경고 발송 페이지**
  - 유저 검색 또는 문제 유저 빠른 선택
  - 개별 발송: 유저 상세 페이지에서 "경고 발송" 버튼
  - 일괄 발송: 체크박스로 복수 선택 후 발송
  - **템플릿 관리**
    - 사전 정의 템플릿 (활동량/고립/비활동/편중)
    - 변수 지원: {username}, {unique_partners}, {login_rate} 등
    - 커스텀 템플릿 생성/수정/삭제
  - **발송 옵션**
    - DM 발송: 유저에게 직접 전송
    - 관리자만 기록: DB에만 저장, DM 미발송
  - **발송 이력**
    - 발송 성공/실패 기록
    - 재시도 옵션

- **자동화**
  - 새벽 4시 벌크 분석 (PostgreSQL 쿼리)
  - **편중 유저만 자동 경고 DM 발송**
  - **고립/비활동 유저는 관리자 웹에만 표시** (수동 판단)
  - user_stats 테이블에 히스토리 저장

### 밴 관리
- **수동 밴 전용** (자동 밴 없음, 관리자 판단 필수)

- **밴 실행 페이지**
  - 유저 검색 및 선택
  - 밴 사유 입력 (필수)
  - 증거물 자동 수집:
    - 경고 이력 (최근 모든 경고)
    - 최신 소셜 통계 (대화 상대, 접속률, 고립/편중/비활동 지표)
    - JSON 형태로 evidence_snapshot에 저장
  - 확인 팝업 (밴 사유 및 증거물 요약 확인)
  - 실행 시 ban_records 테이블에 기록

- **밴 목록 페이지**
  - 밴 유저 목록 조회
  - 필터: 활성/해제
  - 밴 사유, 밴 일시, 관리자명 표시
  - 증거물 조회 버튼 (JSON 포맷팅 모달)

- **언밴 (밴 해제)**
  - 밴 유저 선택
  - 언밴 사유 입력 (필수)
  - is_active=0 처리
  - unbanned_at, unbanned_by, unban_reason 기록

- **증거물 보존**
  - 모든 밴 기록은 영구 보존
  - evidence_snapshot에 경고 이력 + 최신 통계 저장
  - 언밴 후에도 기록 유지

### 툿 아카이빙 (자동)
- **자동 아카이빙 트리거**
  - 경고 횟수가 임계값 이상 (기본: 3회)
  - 경고 발송 시 자동으로 조건 확인
  - 조건 충족 시 자동으로 유저의 모든 툿 아카이빙

- **아카이빙 프로세스**
  - PostgreSQL에서 유저의 모든 statuses 조회
  - SQLite archived_toots 테이블에 저장
  - 저장 항목:
    - 툿 내용 (content)
    - 작성 시각 (created_at)
    - 공개 범위 (visibility)
    - 답글 정보 (in_reply_to_id, in_reply_to_account_id)
    - 미디어 첨부 (media_attachments JSON)
    - 아카이빙 사유 (예: "경고 3회 누적")
    - 당시 경고 횟수
  - 중복 방지: UNIQUE(user_id, toot_id)

- **아카이빙 목록 페이지**
  - 아카이빙된 유저 목록
  - 필터: 날짜 범위, 경고 횟수
  - 정렬: 최근 아카이빙 순
  - 상세 정보:
    - 유저명, 아카이빙 일시
    - 당시 경고 횟수
    - 아카이빙된 툿 개수
    - 아카이빙 사유

- **유저별 아카이빙 조회**
  - 특정 유저의 아카이빙된 모든 툿 조회
  - 필터: 날짜, 공개 범위, 답글 여부
  - 검색: 툿 내용 키워드 검색
  - 툿 상세 보기 (내용, 미디어, 작성 시각)
  - 엑셀 다운로드 (증거 보존용)

- **아카이빙 로그**
  - 아카이빙 실행 이력
  - 성공/실패 기록
  - 실패 시 사유 및 재시도 옵션

### 콘텐츠 관리
**스토리 / 공지 / 운영진 공지 (3개 탭)**

**스토리**:
- 예약 작성 → 스토리 계정으로 발행
- 공개 범위: public/unlisted
- 예약 목록, 발행 내역

**공지**:
- 예약 작성 → 공지 계정으로 발행
- 공개 범위: public/unlisted
- 예약 목록, 발행 내역

**운영진 공지**:
- 예약 작성 → **관리자 봇으로 비공개 툿 발행**
- 관리자들만 팔로우 → 실시간 알림
- 운영진 전용 정보 공유
- 예약 목록, 발행 내역

### 관리 로그
- 전체 작업 기록 조회
  - 필터 (작업 유형별)
  - 기간 선택
  - 검색 (관리자명/유저명)
- 기록 내용
  - 재화 조정
  - 수동 경고 발송
  - 설정 변경
  - 상점 관리 (아이템 추가/수정/삭제)
  - 휴식 등록/해제
- 엑셀 다운로드

## 역할별 기능

### 일반 유저 (role: user)
- 봇 명령어 사용
- 답글로 재화 획득
- 상점에서 구매
- 활동량 체크 대상

### 관리자 (role: admin)
- 봇 명령어 사용
- 답글로 재화 획득
- 상점에서 구매
- **활동량 체크 제외**
- **관리자 웹 접근**

### 시스템 계정 (role: system)
- 봇 명령어 사용 불가
- 재화 획득 안 됨
- **활동량 체크 제외**
- 예: 봇 계정, 공지 계정

### 스토리 계정 (role: story)
- 봇 명령어 사용 가능
- 답글로 재화 획득
- **활동량 체크 제외**
- 예: 스토리 전용 캐릭터

## 알림 시스템

### 관리자 봇 계정
- 비공개 툿으로 알림 발송
- 관리자들만 팔로우
- 활동량 경고 알림
- 시스템 오류 알림 (선택)

### 대시보드 알림
- 경고 대상 유저 수
- 휴식 중인 유저 수
- 봇 상태 이상 시 표시

## 데이터 기록

### transactions (모든 거래)
- earn_reply: 답글로 획득
- earn_attendance: 출석 체크로 획득
- spend_shop: 상점 구매
- admin_add/remove: 관리자 조정
- 추후: 게임 결과, 양도 등

### warnings (경고 기록)
- auto: 자동 경고
- manual: 수동 경고
- 체크 기간, 요구/실제 답글 수
- DM 발송 여부

### admin_logs (관리 작업)
- adjust_balance: 재화 조정
- send_warning: 수동 경고
- register_vacation: 휴식 등록
- change_settings: 설정 변경
- add/edit/delete_item: 상점 관리
- add/edit/delete_event: 일정 관리
- ban_user: 유저 밴
- unban_user: 유저 언밴
- archive_toots: 툿 아카이빙 (자동)

### calendar_events (일정/이벤트)
- 커뮤니티 일정 기록
- 이벤트 타입 (일반/공휴일/공지)
- 전역 휴식기간 표시
- 작성자 기록

# 마스토돈 역할 설정 가이드

> **목적**: CommuManager 대시보드에서 시스템 계정을 필터링하기 위한 마스토돈 역할 설정

---

## 📌 개요

CommuManager는 마스토돈 API를 통해 사용자의 역할 정보를 가져옵니다.
이를 활용하여 대시보드에서 **일반 유저**와 **시스템 계정**(봇, 관리자 등)을 구분할 수 있습니다.

---

## 🎯 권장 역할 구성

마스토돈 서버에 다음 역할을 생성하는 것을 권장합니다:

### 1. **Owner** (기본 제공)
- **설명**: 서버 소유자
- **필터링**: 대시보드 통계에서 **제외**
- **색상**: #ff3838 (빨강)

### 2. **Admin** (기본 제공)
- **설명**: 서버 관리자
- **필터링**: 대시보드 통계에서 **제외**
- **색상**: #ff6600 (주황)

### 3. **Moderator** (기본 제공)
- **설명**: 게시물 관리자
- **필터링**: 대시보드 통계에서 **제외** (선택 사항)
- **색상**: #0099ff (파랑)

### 4. **🤖 봇** (커스텀 역할 - 추가 필요)
- **설명**: 자동화 봇 계정
- **필터링**: 대시보드 통계에서 **제외**
- **권장 색상**: #6c757d (회색)
- **권한**: 없음 (권한 필요 없음, 표시용만)

### 5. **⚙️ 시스템** (커스텀 역할 - 추가 필요)
- **설명**: 시스템 계정 (알림봇, 안내봇 등)
- **필터링**: 대시보드 통계에서 **제외**
- **권장 색상**: #17a2b8 (청록)
- **권한**: 없음

### 6. **🧪 테스트** (커스텀 역할 - 추가 필요)
- **설명**: 테스트용 계정
- **필터링**: 대시보드 통계에서 **제외**
- **권장 색상**: #ffc107 (노랑)
- **권한**: 없음

## 🔧 마스토돈 서버에서 역할 추가하기

### 1단계: 관리자 페이지 접속

```
https://[당신의 마스토돈 서버]/admin/roles
```

예: `https://mastodon.example.com/admin/roles`

### 2단계: 역할 생성

1. **"새 역할 추가"** 버튼 클릭
2. 역할 정보 입력:
   - **이름**: `봇` (또는 원하는 이름)
   - **색상**: `#6c757d` (회색)
   - **강조 표시**: ✅ 체크 (프로필에 표시됨)
   - **권한**: 체크 안 함 (권한 불필요)

3. **저장** 클릭

### 3단계: 사용자에게 역할 할당

1. **서버 설정** → **역할** 탭 이동
2. 방금 만든 역할 (`봇`) 클릭
3. **사용자 추가** 버튼 클릭
4. 봇 계정 검색 후 선택
5. **저장**

## 📊 CommuManager에서 활용 방법

### 자동 필터링

CommuManager는 다음 역할을 가진 사용자를 **대시보드 통계에서 자동 제외**합니다:

```python
SYSTEM_ROLES = ['Owner', 'Admin', 'Moderator', '봇', '시스템', '테스트']
```

### 역할 정보 동기화

- **자동 동기화**: 사용자가 로그인할 때마다 마스토돈 API로 역할 정보를 가져옵니다
- **수동 업데이트 불필요**: 마스토돈에서 역할을 변경하면 다음 로그인 시 자동 반영

## 🔍 역할 확인하기

마스토돈 API로 역할을 제대로 가져오는지 확인:

```bash
# 사용자 정보 조회 (access_token 필요)
curl -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
     https://mastodon.example.com/api/v1/accounts/verify_credentials
```

**응답 예시:**
```json
{
  "id": "123456",
  "username": "bot_account",
  "role": {
    "id": "5",
    "name": "봇",
    "color": "#6c757d"
  }
}
```

## ⚠️ 주의사항

1. **역할 강조 표시 필수**
   - 역할 생성 시 "강조 표시" 옵션을 **반드시 체크**해야 합니다
   - 체크하지 않으면 API 응답에 역할 정보가 포함되지 않습니다

2. **마스토돈 버전 확인**
   - 역할 API는 Mastodon v4.0.0 이상에서 사용 가능합니다
   - 버전 확인: `https://[서버]/api/v1/instance`

3. **원격 계정**
   - 다른 서버의 계정(원격 계정)은 역할 정보가 없을 수 있습니다
   - CommuManager는 로컬 계정만 역할 필터링을 적용합니다
